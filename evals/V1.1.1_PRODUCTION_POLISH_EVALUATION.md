## What changed (8 polish areas)

### 1. SEO — Development vs Professional

| Mode | v1.1.0 behavior | v1.1.1 behavior | Eval result |
|------|-----------------|-----------------|-------------|
| **Development** (default) | Async job; librecrawl stall on localhost | Inline `status: completed`, `instant: true`, browser-only providers | ✅ No librecrawl stall; audit_id returned |
| **Professional** | Async enqueue ~90ms | Same async path; auth-gated without OAuth | ✅ `auth_required` in 0.6ms (expected) |
| **Localhost detection** | Full provider stack | Forces development unless `mode=professional` | ✅ `http://localhost:5173` used dev path |

**Development SEO measurement**

| Metric | Target | Measured | Pass |
|--------|--------|----------|------|
| Blocking | No background job | `instant: true`, no `audit_job_id` | ✅ |
| Status | `completed` inline | `status: completed` | ✅ |
| Latency | <2s | **4,292ms** | ❌ |
| Providers | No crawl/auth on localhost | No librecrawl progress stall | ✅ |

**Root cause of 4.3s:** Logs show `code_review_graph` FTS rebuild during audit when `repo_root` is set (Sprint 2 codebase hints). Browser + AI visibility path is fast; graph indexing dominates.

**Recommendation:** Skip or defer CRG indexing in Development SEO mode, or run hints in `asyncio.to_thread` with a 1.5s cap.
