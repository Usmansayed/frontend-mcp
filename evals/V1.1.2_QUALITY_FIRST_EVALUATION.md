# v1.1.2 Quality-First Evaluation (pre-release)

**Date:** 2026-07-13  
**Baseline:** v1.1.1 installed E2E (`evals/V1.1.1_END_USER_MCP_EVALUATION.md`, score 8.2/10)  
**Method:** Handler-layer polish eval + focused pytest + stdio MCP resource wire test  
**Artifact:** `artifacts/evals/v1.1.1_polish_eval.json` (post-fix run)

## Executive summary

**Verdict: Quality-first fixes are ready for v1.1.2 publish after MCP restart + Cursor resource smoke test.**

The v1.1.1 polish pass over-optimized two areas at the expense of usefulness:

1. **Development SEO** — hard 2s `ok:false` timeout sacrificed recommendations for a benchmark.
2. **Component search** — `max_registries=8` and 1.5s doc timeout reduced match breadth.

Additionally, **MCP resources were broken at the protocol layer** — `read_resource` returned `TextResourceContents` objects instead of `ReadResourceContents`, causing `resources/read` to fail in all MCP clients.

## Quality vs latency audit

| Optimization (v1.1.1) | Quality impact | v1.1.2 fix |
|----------------------|----------------|------------|
| Development SEO 2s hard fail | **High** — zero recommendations on timeout | 5s budget; skip CRG + fast codebase in dev; `ok:true` partial audit |
| `max_registries=8` | **Medium** — fewer shadcn registry hits | Restored to 12 (3–5s search acceptable) |
| Integrate doc timeout 1.5s | **Medium** — fast fallback docs | 3s doc read; 8s/12s handler budgets |
| `summary_only` default observe | **None** — evidence collected fully; serialization trimmed only | Kept |
| `plan_only` default integrate | **Low** — by design; execute path still available | Kept; longer budgets when executing |
| Warm shadcn cache | **None** — improves latency without changing scoring | Kept |
| AI reasoning auto in dev SEO | **Low/Medium** — non-deterministic latency | Disabled by default in development mode |
| CRG in dev SEO with `repo_root` | **High latency** — 4s+ graph search | Skipped in development mode |

## Fixes implemented

### 1. Development SEO (usefulness-first)

- `development_audit_bounded(budget_s=5)` with partial fallback (`development_seo_partial_budget`)
- Development mode skips CRG (`include_crg=False`), uses fast targeted codebase globs (no full `rglob`)
- AI reasoning off by default in development unless explicitly requested
- Handler returns `ok:true` with `status: partial` instead of `development_seo_timeout` error

**Result:** 2,797ms, `status: completed`, 31KB payload with recommendations (was 2,050ms `ok:false` on v1.1.1).

### 2. MCP resources (protocol bug)

- Root cause: MCP SDK expects `ReadResourceContents(content=..., mime_type=...)` from handlers, not `types.TextResourceContents`
- Fixed `read_resource_handler` in `server.py`
- Added `list_resource_templates` for `perception://scan/{scan_id}/{artifact}`
- Stdio test: `resources/read` returns 21K+ char agent guide

**Cursor verification:** Requires MCP server restart on local branch / v1.1.2 install — not testable in this session (`user-frontend-perception` server unavailable).

### 3. Coordination layer

- **Scan reuse:** `_maybe_advance_cached_observe` skips redundant observe steps when `scan_id` exists and `ui_runtime` posture ≥ partial
- **Anti-patterns wired:** `no_scan_id_and_no_snapshot_id` blocks design capabilities; `same_page_rapid_iteration` defers heavy audits after 2+ observes with same scan
- Advisory only for routing — execution-runtime idempotency unchanged

### 4. Component quality restoration

- `max_registries`: 8 → 12
- Doc timeout: 1.5s → 3s
- Integrate handler: 8s plan / 12s execute (was 3s / 4.5s)
