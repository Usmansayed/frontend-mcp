## Executive summary

**Verdict: Production-usable for core browser loops, resolver intelligence, and fast resource/icon workflows. Not yet reliable for component integration, full SEO async completion on localhost, or MCP resource reads in Cursor.**

v1.1.0 delivers the headline features from this release: **sub-200ms resolvers** (route, component, claim validation, live correlate) and **non-blocking SEO job enqueue** (~90ms). Core browser observe → probe → verify remains strong. Regressions and carry-overs from v1.0.1: **`perception_integrate_component` 60s timeout**, **component search ~25s**, **SEO job stuck at 10% on librecrawl**, **MCP resources not fetchable**, **font search HTTP 400**.

## Recommended improvements

### Release-blocking (v1.1.1)

1. **Component integrate timeout** — Profile orchestration pipeline; add fast-fail when shadcn registry fetch blocks; return partial plan under 5s with `degraded: ["integration_timeout"]` instead of 60s hard timeout.
2. **SEO localhost profile** — Skip or short-circuit `librecrawl`/`lighthouse` for `localhost`/`127.0.0.1`; use browser-only evidence path in development mode.
3. **Guard probe session hygiene** — Save URL before maze run; restore after; or return `data.final_url` + `degraded: ["session_url_changed"]`.

### High value (v1.2)

4. **Component search cache** — Warm shadcn catalog at MCP startup; target <2s for common queries (date picker, calendar, button).
5. **MCP resource registration** — Fix Cursor compatibility for `perception://agent-guide`, `perception://resolver-guide`, `perception://seo-guide`.
6. **`summary_only` observe mode** — Document and default for re-observe after first full scan; cut payload 80%+.
7. **Fontsource API** — Fix query encoding / endpoint for v1 Fontsource API.

### Quality / DX

8. **Schema examples in tool descriptions** — Embed minimal JSON examples for `claim`, `routes[]`, `scan_id_before/after`.
9. **Resolver plugins** — Next.js App Router, Remix route plugins (documented roadmap).
10. **Execution tier visibility** — Expose `tier` in execution block so agents know BACKGROUND vs FAST paths.
