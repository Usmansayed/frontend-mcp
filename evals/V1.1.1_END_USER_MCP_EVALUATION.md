## Executive summary

**Verdict: Production-usable for core browser loops, resolvers, component search/integrate, and lightweight observe. Two polish gaps remain on the installed package: Development SEO hits the hard 2s timeout, and Cursor still cannot fetch `perception://*` resources.**

v1.1.1 delivers the headline polish wins: **`perception_integrate_component` 1.7s** (was 60s timeout), **`perception_search_components` ~1s** (was 25.6s), **`probe_guards` restores URL**, **`navigate_and_observe` defaults to `summary_only`** (no DOM in envelope). Development SEO no longer stalls on librecrawl, but the **2.0s handler cap rejects audits that need ~2.0тАУ4.3s** to complete.

| Tool | Result | Notes |
|------|--------|-------|
| `perception_seo_status` | тЬЕ | Development mode shows `providers: ["browser"]`, `instant: true` |
| `perception_seo_audit_start` (dev + `scan_id`) | тЭМ | `error: development_seo_timeout` at 2,050ms (with and without `repo_root`) |

**Analysis:** The 2.0s `asyncio.wait_for` cap in the handler fires before `development_audit` completes on the installed MCP process. Handler-layer eval completed in 4.3s with `repo_root`; without cap it works but exceeds the advertised &lt;2s target.

**Workaround for agents:** Use `perception_seo_audit` legacy path or increase dev timeout in v1.1.2; omit `repo_root` does not fix timeout in this run.
